# Tiny Cowork v2.2 성능 및 포맷 요구사항

사용자의 최근 피드백을 바탕으로 성능 최적화 및 출력 포맷을 다음과 같이 정의합니다.

## 1. 데이터 포맷 요구사항
- **검색 결과**: 기존과 동일하게 명시적인 URL 텍스트를 포함해야 함. (`* 제목 - 요약 (링크: URL)`)
- **날씨 결과**: 날씨 정보는 온도가 가장 중요하므로 온도를 명확히 표기하고, 검색 결과와 달리 출처 URL은 포함하지 않음.
  - **포맷**: `* 도시명 날씨 - 온도/상태` (예: `* 서울 날씨 - -4°C, 맑음`)

## 2. 성능 최적화 (보고서 생성 속도)
- **문제점**: Thinking 모델의 사고 과정(Thinking traces)으로 인해 최종 리포트 생성이 너무 오래 걸림.
- **해결책**: 최종 리포트 생성 단계(`integrate_response`)에서는 Thinking 기능을 비활성화함.
  - **실행 방식**: Thinking 모델의 가중치는 그대로 사용하되, 생성 파라미터를 `Instruct` 모델 수준(`temperature` 등)으로 조정하고 사고 과정을 유도하지 않음.
  - **기대 효과**: 리포트 정확도는 유지하면서 생성 속도를 획기적으로 개선.

## 3. 언어 처리 전략 (English-First)
- **방향성**: 모델의 지능을 최대화하기 위해 내부 처리는 모두 영어로 진행.
  1. 사용자 질문을 영어로 번역 (Reasoner/Planner 단계).
  2. Brain 모델은 영어로 최종 리포트를 상세하게 작성.
  3. 작성된 영어 리포트를 구글 번역기(또는 파이프라인)를 통해 한국어로 번역하여 최종 출력.
- **주의사항**: 번역 과정에서 내용이 누락되거나 과하게 요약되지 않도록 프롬프트를 정교화할 것.
